# Coding Standards

## General Principles
- Write clear, readable, and maintainable code.
- Favor consistency over cleverness.
- Keep functions focused and responsibilities well defined.

## Formatting
- Follow language-specific formatting tools and linters.
- Use descriptive naming for variables, functions, and files.
- Keep comments purposeful and concise.

## Quality Expectations
- Add tests for business logic and critical workflows.
- Avoid duplicate logic and prefer reusable abstractions.
- Review code for security, reliability, and readability.

## Version Control Expectations
- Make small, reviewable changes.
- Include meaningful commit messages.
- Link work items or tickets in pull requests where applicable.
